> **⚠️ Historical snapshot — captured AT AUDIT TIME.** This documents the old, broken **name-string** model as it stood before any fixes. It is preserved as-is on purpose and is **no longer the current model**: linking is now **id-based** and the relationships below have been corrected (Company as the hub; `companyId` / `primaryContactId` / `additionalContactIds`; rename-safe joins; leads/deals separated). For how the model works today, see **`_ARCHITECTURE.md`** and **`changes/id-based-linking.md`**.

This is the entity-relationship map **as the prototype actually implements it today** — not how it should be. Integrity flags: **clean** = solid link · **fragile** = works but breaks easily (e.g. matched by a copied name string) · **broken** = wrong or non-functional.
